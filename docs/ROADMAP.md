## 1. 완료된 주요 기능 (현재 라이브)

- ✅ MD 업로드 → 렌더링 페이지 + 공유 URL 생성
- ✅ GFM · 코드 구문 강조 · TOC · LaTeX 수식 · Mermaid 다이어그램 렌더링
- ✅ View / Edit 모드, Edit 실시간 프리뷰, 서버 저장
- ✅ Export (Markdown / HTML / PDF)
- ✅ 다크 / 라이트 모드 (v0.2, 2026-03-24)
- ✅ 포인트 컬러 디자인 (`#FFF7E6`, `#0A122A`)
- ✅ **HTML 문서 지원** — 자기 완결형(인라인 CSS/JS/data-URI, 중첩 iframe) HTML을 샌드박스 iframe으로 렌더링
- ✅ **본문 R2 저장 전환 + 업로드 한도 25MB** (2026-07-22) — 본문을 Cloudflare R2에 저장(D1은 메타데이터만)해 D1 행당 ~2MB 한계를 회피. MD·HTML 공통 25MB
- ✅ 브라우저 확장 프로그램 — Chrome/Edge 사이드 패널 렌더링, 로컬 `.md` file:// 리다이렉트, 원격 `.md` 인터셉트
- ✅ **모바일 헤더 반응형(햄버거) + 가로 오버플로 버그 수정** (2026-07-22) — 좁은 화면에서 헤더 컨트롤이 뷰포트 폭을 넘쳐 가로 스크롤바와 우측·하단 여백이 생기던 버그 해결. MD/HTML 뷰 공통 헤더를 `EditorHeader`로 추출하고, `md` 미만에서 저장·Export·공유·테마·스크롤 동기화를 햄버거 드롭다운으로 수납 + 루트에 `overflow-x` 가드 추가

## 2. 고도화 백로그 — 웹 앱

| 우선순위 | 항목 | 설명 / 메모 |
|:---:|------|------|
| P1 | **markview MCP 서버 (AI 산출물 즉시 발행)** | 요즘 HTML/MD는 대부분 Claude 등 AI 산출물 → markview를 MCP 서버로 만들어 Claude에 연결. Claude가 만든 HTML/MD를 MCP `publish` 도구로 바로 공유 링크(`/v/{slug}`)화하고 웹에서 렌더링 열람. **기존 `/api/upload`(CORS 오픈) 재사용 → 얇은 래퍼로 저비용.** 배포는 원격 MCP(Cloudflare Workers) 권장. **단계: ① 인증 없는 공개 `publish` MVP → ② 계정 도입 후 OAuth 연결해 "내 문서함" 자동 연동** (2026-07-22 도출) |
| P1 | **멀티파일 HTML 사이트 지원** | `index.html` + 별도 CSS/JS/이미지/다른 HTML 페이지 참조 구조. 현재는 단일 파일(srcDoc)만 되어 상대경로 에셋이 404. 폴더/ZIP 업로드 → R2에 `{slug}/경로` 저장 → 실제 URL 서빙(iframe `src` 전환) 필요. (2026-07-22 신규 도출) |
| P1 | **이미지 업로드** | 현재 외부 URL만 지원. **R2 인프라가 이미 붙어 있어 착수 부담이 낮음** — 본문 저장에 쓰는 `CONTENT` 버킷 활용 가능 |
| P1 | 문서 삭제 / 만료 기능 | 삭제 API 미구현 상태. 업로드 시 만료기간(7일·30일 등) 옵션 + 직접 삭제 |
| P2 | 사용자 인증 / 로그인·회원가입 | **"내 문서 목록"의 선행 조건.** 기기 종속을 배제(다른 PC·폰에서도 동일 목록)하려면 localStorage/디바이스 토큰으로는 불가 → 계정 체계 필수로 확정 (2026-07-22 결정) |
| P2 | 내 문서 목록 관리 | 폴더 · 태그 · 검색. **인증 선행 필요** (기기 간 동기화 목록) |
| P2 | 접근 제어 | 비공개 문서, 링크 비밀번호, 조직 내부 공유 제한 |
| P2 | **HWP/HWPX ↔ MD 변환** | HWP/HWPX 업로드 → 마크다운 자동 변환(편집·공유 원스톱), MD → HWP/HWPX Export. 국가AI전략위 공공문서 마크다운 전환 정책(2026-03-05) 대응, 한컴 파서 기술 활용 — **전략 항목** |
| P2 | 공개 API 업로드 | 확장이 이미 `/api/upload` 사용 중 → 외부 공개 API로 정식화 |
| P3 | 문서 버전 히스토리 | |
| P3 | 실시간 협업 편집 | |
| P3 | 커스텀 도메인 매핑 | |
| P3 | 단축 URL / QR 코드 | |
| P3 | OG 이미지 자동 생성 | |
| P3 | 외부 콘텐츠 크래퍼 | |
| P3 | Google Chat 봇 연동 | `.md` 첨부 시 Markview URL 자동 생성·회신 |
| P3 | Google Workspace 연동 강화 | Google Drive 직접 열람, Gmail 첨부 인터셉트 |
| P3 | 팀 / 조직 워크스페이스 | 팀 공유 문서 목록·검색·분류 |

## 4. 현재 제약 사항 (스냅샷)

**웹 앱**

| 항목 | 현황 |
|------|------|
| 인증 | 없음 — URL을 아는 누구나 열람/편집 |
| 문서 삭제 | API 미구현 |
| 문서 목록 | 없음 — slug를 알아야 접근 |
| 파일 크기 | **MD·HTML 공통 25MB** (2026-07-22 상향, 이전 512KB/2MB) |
| 이미지 호스팅 | 미지원 — 외부 URL만 (R2 인프라는 확보됨) |
| HTML 멀티파일 | 미지원 — 단일 자기완결형 파일만 |
| 문서 만료 | 없음 — 영구 보관 |
| 모바일 | 헤더 햄버거 반응형 적용 완료(2026-07-22). Edit 모드 Split View 모바일 UX는 여전히 미최적화 |

**확장 프로그램**

| 항목 | 현황 |
|------|------|
| 브라우저 지원 | Chrome 114+ / Edge 114+ (Chromium Side Panel API) |
| 오프라인 | 렌더링 가능, 웹 공유 불가 |
| 이미지 | 외부 URL만, 로컬 이미지 미지원 |
| Edit 모드 | 단일 textarea (웹 앱과 차이) |
| TOC | 사이드 패널 크기 제약으로 미제공 |
| 인터셉트 | 인증 필요 사이트에서 fetch 실패 가능성 |
