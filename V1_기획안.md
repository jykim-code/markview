# Markview V1 기획안

> Markdown + View, 보기 좋게 렌더링
> 작성일: 2026-03-18 | V1 완료: 2026-03-19 | v1.1 다크모드: 2026-03-24

---

## 1. 서비스 한 줄 소개

**마크다운 파일을 업로드하면 아름답게 렌더링된 웹 페이지와 공유 URL을 즉시 제공하는 서비스**

### 핵심 차별점

> 다른 마크다운 뷰어도 렌더링·편집·내보내기를 지원하지만, **Markview의 진짜 장점은 "웹 공유"**에 있다.
> URL 하나만 전달하면 수신자는 별도 설치나 파일 다운로드 없이 **누구나 아름답게 렌더링된 문서를 즉시 열람**할 수 있다.

## 3. 핵심 플로우

```
사용자 접속 → MD 파일 업로드 → View 페이지에서 렌더링 확인 → 공유 URL 복사
                                                        → Edit 모드 전환하여 편집
```

### 3.1 메인 페이지 (업로드)
- 중앙에 파일 업로드 영역 (드래그 앤 드롭 / 클릭)
- `.md` 파일만 허용 (최대 512KB)
- 업로드 즉시 **View 페이지**로 이동
- 반응형 레이아웃: 모바일은 세로 배치, 데스크톱은 가로 배치
- 헤더 네비게이션: Home, About, Privacy, Contact, 다크모드 토글

### 3.2 뷰 페이지 (렌더링 + 편집 + 공유)
- 경로: `/v/{slug}` (View와 Edit가 하나의 페이지에 통합)
- **View / Edit 모드 전환** 토글 (라운드 pill 형태)
  - **View 모드** (기본): 렌더링된 문서 + TOC 사이드바 (데스크톱)
  - **Edit 모드**: 좌측 마크다운 에디터 + 우측 실시간 프리뷰
- **마크다운 복사**: Edit 모드에서 Markdown 라벨 영역의 복사 아이콘 클릭 시 마크다운 원본 텍스트를 클립보드에 복사 (체크 아이콘 피드백)
- **저장 (`.md 저장`)**: 편집한 마크다운을 서버에 PUT 요청으로 저장, 저장 완료 시 "저장 완료!" 피드백
- **Export**: Markdown(`.md`) / HTML(`.html`) / PDF 세 가지 형식으로 내보내기
  - PDF는 print CSS로 렌더링된 콘텐츠만 출력 (헤더/에디터/사이드바 제외)
- **공유** (네이비 Primary CTA): 현재 문서의 공유 URL을 클립보드에 복사

| 문법 | 지원 |

| 이미지, 링크 | ✅ |

| LaTeX 수식 | ✅ `$inline$`, `$$block$$` (KaTeX) |
| Mermaid 다이어그램 | ✅ `mermaid` 코드 블록 |

## 6. 데이터 구조

### Document
| 필드 | 타입 | 설명 |
|------|------|------|
| id | TEXT (UUID) | 고유 식별자 |
| slug | TEXT (UNIQUE) | URL용 짧은 ID (8자 UUID prefix) |

| created_at | TEXT | 생성 시각 (datetime) |

## 7. 페이지 구성

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 메인 (업로드) | `/` | 히어로 + 업로드 UI + What We Support |
| 뷰/편집 | `/v/{slug}` | View/Edit 모드 통합, 저장, Export, 공유 |
| About | `/about` | 서비스 소개, 주요 기능, 운영자 정보 |
| Privacy | `/privacy` | 개인정보 처리방침 |
| Contact | `/contact` | Formspree 문의 폼 |
| 404 | `/v/{없는slug}` | 문서를 찾을 수 없음 안내 |
| 전역 404 | `/*` | 페이지를 찾을 수 없음 안내 |

## 9. 모바일 반응형

- 메인 히어로: `flex-col` (모바일) → `flex-row` (데스크톱)
- 폰트: `clamp(32px, 6vw, 52px)` 반응형 타이틀
- 뷰 페이지: 패딩 축소 (`px-5` / `md:px-12`), 타이틀 `28px` → `md:40px`
- TOC 사이드바: `lg:block` (데스크톱에서만 표시)
- 모든 헤더/푸터: 모바일 패딩 대응

## 10. 다크모드 지원

> 추가일: 2026-03-24

### 구현 방식
- `data-theme` 속성 기반 CSS 변수 전환 (라이트/다크)
- `localStorage`에 테마 저장, 새로고침 시 유지
- OS 시스템 설정(`prefers-color-scheme`) 자동 감지 (초기값)
- FOUC 방지: `<head>` 인라인 스크립트로 렌더링 전 테마 적용

### 적용 범위

| 로고 | ✅ 라이트/다크 SVG 분리 (`logo-light` / `logo-dark` 클래스) |

| Mermaid 다이어그램 | ✅ 다크 테마 변수 적용 |
| KaTeX 수식 | ✅ 텍스트 색상 전환 |
| 카드/그림자 | ✅ `--shadow-card` 다크 전용 값 |
| 에디터/프리뷰 | ✅ 보더, 라벨 색상 전환 |
