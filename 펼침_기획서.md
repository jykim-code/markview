## 1. 서비스 개요

### 1.1 서비스 소개
- **서비스명**: 펼침 (Pyulchim)
- **한 줄 소개**: 마크다운 문서를 아름다운 웹 페이지로 펼쳐서 공유하는 서비스
- **서비스 목적**: 마크다운 파일을 업로드하면 즉시 공유 가능한 URL을 생성하여, 누구나 보기 좋게 렌더링된 문서를 열람할 수 있도록 함

### 1.2 해결하고자 하는 문제

**현재 문제점:**
- 마크다운 파일을 비개발자에게 공유하기 어려움 (원본 그대로 보내면 문법이 그대로 노출)
- Obsidian, Notion 등에서 작성한 문서를 외부에 공유하려면 별도 설정이 필요
- 기존 마크다운 공유 서비스들은 디자인이 투박하거나 커스터마이징이 제한적

**펼침의 해결책:**
- 마크다운 파일을 업로드하면 자동으로 예쁘게 렌더링된 웹 페이지 생성
- 공유 URL 하나로 누구나 즉시 열람 가능
- 미니멀하고 가독성 높은 디자인으로 문서의 품격을 높임

## 2. 타겟 사용자

### 2.1 주요 사용자
- 마크다운으로 문서를 작성하는 개발자, 기획자, 작가
- 기술 문서, 가이드, 보고서 등을 외부에 깔끔하게 공유하고 싶은 사람
- Obsidian, VS Code 등에서 마크다운을 주로 사용하는 사람

### 2.2 사용자 유형별 정의

| 사용자 유형 | 설명 | 주요 권한/기능 |
|------------|------|---------------|
| 관리자 | 서비스 운영자 (1명) | 문서 업로드, 관리, 삭제, 폴더/태그 관리, 비밀번호 설정 |
| 뷰어 | 공유 링크를 받은 사람 | 문서 열람만 가능 (비밀번호 입력 필요 시 입력 후 열람) |

### 3.2 기능 상세

#### 3.2.1 마크다운 업로드
- **설명**: 관리자가 .md 파일을 웹 인터페이스에서 업로드
- **동작 방식**: 
  - 파일 선택 또는 드래그 앤 드롭으로 업로드
  - 단일 파일 또는 여러 파일 동시 선택 가능 (배치 업로드)
- **입력**: .md 파일 (1개 이상)
- **출력/결과**: 
  - 문서가 GitHub 저장소에 커밋됨
  - 문서 내 로컬 이미지는 AWS S3에 업로드 후 URL로 변환됨
  - 관리자 페이지 문서 목록에 추가됨
- **예외 상황**: 
  - .md 외 파일 업로드 시 오류 메시지 표시
  - 동일 파일명 존재 시 덮어쓰기 여부 확인

#### 3.2.2 마크다운 렌더링
- **설명**: 마크다운 문법을 HTML로 변환하여 예쁘게 표시
- **동작 방식**: 
  - 표준 마크다운 문법 전체 지원
  - Obsidian frontmatter (YAML 메타데이터) 파싱 및 표시
  - 코드 블록 구문 강조 (Syntax Highlighting)
  - 유튜브, 트위터 등 외부 콘텐츠 임베드 지원
  - 이미지, 비디오 렌더링
  - 테이블, 체크리스트, 인용구 등 확장 문법 지원
- **입력**: 저장된 마크다운 문서
- **출력/결과**: 가독성 높은 HTML 페이지
- **예외 상황**: 지원하지 않는 문법은 원본 텍스트로 표시

#### 3.2.3 공유 URL 생성
- **설명**: 업로드된 문서마다 고유한 공유 URL 생성
- **동작 방식**: 
  - 업로드 완료 시 자동으로 URL 생성
  - URL 형식: `https://[도메인]/docs/[문서ID]`
  - 관리자 페이지에서 COPY 버튼 클릭 시 클립보드에 복사
- **입력**: 업로드된 문서
- **출력/결과**: 공유 가능한 고유 URL
- **예외 상황**: 삭제된 문서 URL 접근 시 404 페이지 표시

#### 3.2.4 단축 URL 생성
- **설명**: 긴 공유 URL을 짧은 형태로 변환
- **동작 방식**:
  - 문서 카드에서 "단축 URL" 버튼 클릭
  - 단축 URL 형식: `https://[도메인]/s/[짧은ID]` (예: `/s/abc123`)
  - 단축 URL 클릭 시 원본 URL로 리다이렉트
  - 클립보드에 자동 복사
- **입력**: 원본 공유 URL
- **출력/결과**: 단축된 URL
- **예외 상황**: 단축 URL도 문서 삭제 시 함께 무효화

#### 3.2.5 QR 코드 생성
- **설명**: 공유 URL을 QR 코드 이미지로 생성
- **동작 방식**:
  - 문서 카드에서 "QR" 버튼 클릭
  - QR 코드 모달 표시 (미리보기)
  - 옵션:
    - **이미지 다운로드**: PNG 파일로 다운로드
    - **이미지 복사**: 클립보드에 이미지 복사
    - **크기 선택**: 소(128px) / 중(256px) / 대(512px)
- **입력**: 공유 URL (원본 또는 단축 URL 선택 가능)
- **출력/결과**: QR 코드 이미지
- **예외 상황**: 복사 실패 시 다운로드 권유

#### 3.2.6 이미지 자동 처리
- **설명**: 마크다운 내 로컬 이미지 경로를 감지하여 S3에 업로드 후 URL 변환
- **동작 방식**:
  - 마크다운 파일 파싱 시 이미지 경로 추출 (`![](./images/photo.png)` 등)
  - 로컬 경로인 경우 해당 이미지 파일을 함께 업로드하도록 요청
  - 이미지를 AWS S3에 업로드
  - 마크다운 내 이미지 경로를 S3 URL로 자동 변환
- **입력**: 마크다운 파일 + 이미지 파일들
- **출력/결과**: 이미지 URL이 S3 경로로 변환된 마크다운
- **예외 상황**: 
  - 이미지 파일 누락 시 경고 메시지 표시 (업로드는 진행)
  - 지원하지 않는 이미지 형식 안내

#### 3.2.7 폴더 관리
- **설명**: 문서를 폴더별로 분류하여 관리
- **동작 방식**:
  - 좌측 사이드바에 폴더 목록 표시
  - "+" 버튼으로 새 폴더 생성
  - 폴더 클릭 시 해당 폴더의 문서만 필터링
  - "전체" 선택 시 모든 문서 표시
  - **기본 폴더 "Inbox"**: 폴더 미지정 시 자동으로 Inbox에 저장
  - 문서 업로드 시 폴더 지정 가능 (미지정 시 Inbox)
- **입력**: 폴더명
- **출력/결과**: 폴더 생성 및 문서 분류
- **예외 상황**: 동일 폴더명 존재 시 중복 안내, Inbox 폴더는 삭제 불가

#### 3.2.8 폴더/태그 수정
- **설명**: 업로드된 문서의 폴더 이동 및 태그 수정
- **동작 방식**:
  - 문서 카드에서 "수정" 버튼 클릭 (또는 문서 제목 옆 편집 아이콘)
  - 수정 모달 표시:
    - 폴더 변경: 드롭다운에서 다른 폴더 선택
    - 태그 수정: 태그 추가/제거
  - 저장 시 즉시 반영
- **입력**: 변경할 폴더, 태그
- **출력/결과**: 문서 메타데이터 업데이트
- **예외 상황**: 존재하지 않는 폴더/태그 선택 불가

#### 3.2.9 태그 관리
- **설명**: 문서에 태그를 부여하여 분류 및 필터링
- **동작 방식**:
  - 좌측 사이드바 TAGS 영역에 태그 목록 표시
  - "+" 버튼으로 새 태그 생성 (색상 지정 가능)
  - 태그 클릭 시 해당 태그가 부여된 문서만 필터링
  - 문서 업로드 또는 수정 시 태그 지정 가능 (복수 선택)
  - 상단 필터 버튼으로 빠른 태그 필터링 (전체, 직접, 링크 등)
- **입력**: 태그명, 색상
- **출력/결과**: 태그 생성 및 문서 분류
- **예외 상황**: 동일 태그명 존재 시 중복 안내

#### 3.2.7 검색
- **설명**: 문서 제목 또는 내용에서 키워드 검색
- **동작 방식**:
  - 상단 검색창에 키워드 입력
  - 실시간 또는 엔터 입력 시 검색 실행
  - 문서 제목과 내용에서 매칭되는 문서 필터링
- **입력**: 검색 키워드
- **출력/결과**: 매칭되는 문서 목록
- **예외 상황**: 검색 결과 없을 시 "검색 결과가 없습니다" 표시

#### 3.2.11 날짜 필터
- **설명**: 시작일과 종료일을 지정하여 문서 필터링
- **동작 방식**:
  - 상단에 시작일, 종료일 날짜 선택기 제공
  - 날짜 범위 내에 생성된 문서만 표시
  - 필터 초기화 버튼으로 전체 문서 보기
- **입력**: 시작일, 종료일
- **출력/결과**: 해당 기간 내 생성된 문서 목록
- **예외 상황**: 시작일이 종료일보다 늦으면 오류 안내

#### 3.2.12 문서 잠금 (비밀번호 설정)
- **설명**: 특정 문서에 비밀번호를 설정하여 열람 제한
- **동작 방식**:
  - 문서 목록에서 잠금 아이콘(🔒) 클릭
  - 비밀번호 입력 모달 표시
  - 비밀번호 설정 완료 시 잠금 아이콘 활성화
  - 잠긴 문서의 공유 URL 접근 시 비밀번호 입력 화면 표시
- **입력**: 문서별 비밀번호
- **출력/결과**: 문서 잠금 상태 변경
- **예외 상황**: 비밀번호 미입력 시 잠금 설정 불가

#### 3.2.13 비밀번호 입력 (뷰어)
- **설명**: 잠긴 문서 열람 시 비밀번호 입력
- **동작 방식**:
  - 잠긴 문서 URL 접근 시 비밀번호 입력 화면 표시
  - 올바른 비밀번호 입력 시 문서 열람
  - 세션 동안 비밀번호 유지 (재입력 불필요)
- **입력**: 비밀번호
- **출력/결과**: 문서 열람 또는 오류 메시지
- **예외 상황**: 비밀번호 오류 시 "비밀번호가 틀렸습니다" 표시

#### 3.2.14 관리자 로그인
- **설명**: 관리자 페이지 접근을 위한 인증
- **동작 방식**:
  - 관리자 페이지 URL 접근 시 로그인 화면 표시
  - 아이디/비밀번호 입력 후 로그인
  - 로그인 성공 시 관리자 대시보드 표시
  - LOGOUT 버튼으로 로그아웃
- **입력**: 아이디, 비밀번호
- **출력/결과**: 관리자 세션 생성
- **예외 상황**: 인증 실패 시 오류 메시지 표시

#### 3.2.15 문서 삭제
- **설명**: 업로드한 문서 삭제
- **동작 방식**:
  - 문서 목록에서 DELETE 버튼 클릭
  - 삭제 확인 모달 표시
  - 확인 시 GitHub 저장소에서 파일 삭제
  - 관련 이미지도 S3에서 삭제
- **입력**: 삭제할 문서 선택
- **출력/결과**: 문서 및 관련 이미지 삭제
- **예외 상황**: 삭제 실패 시 오류 메시지 표시

#### 3.2.16 API 업로드
- **설명**: API를 통해 프로그래매틱하게 문서 업로드
- **동작 방식**:
  - REST API 엔드포인트 제공 (`POST /api/upload`)
  - API 키 인증 (헤더: `Authorization: Bearer {API_KEY}`)
  - 마크다운 파일, 이미지, 첨부파일을 멀티파트로 전송
  - 폴더, 태그 지정 가능 (요청 바디에 포함, 미지정 시 Inbox)
  - 업로드 성공 시 공유 URL 반환
- **입력**: API 키, 마크다운 파일, 이미지 파일들, 첨부파일들, 폴더명(선택), 태그(선택)
- **출력/결과**: 
  ```json
  {
    "success": true,
    "url": "https://pyulchim.com/docs/abc123",
    "shortUrl": "https://pyulchim.com/s/abc123",
    "documentId": "abc123",
    "createdAt": "2025-02-04T12:00:00Z"
  }
  ```
- **예외 상황**: 인증 실패(401), 파일 형식 오류(400), 서버 오류(500) 시 에러 응답

#### 3.2.17 첨부파일 업로드
- **설명**: 마크다운과 함께 다운로드 가능한 첨부파일 업로드
- **동작 방식**:
  - 문서 업로드 시 첨부파일 함께 선택
  - 첨부파일은 AWS S3에 저장
  - 마크다운 내 `::file-download` 디렉티브로 다운로드 링크 삽입
  - 지원 형식: zip, pdf, docx, xlsx, pptx 등
- **입력**: 첨부파일 (1개 이상)
- **출력/결과**: S3 URL로 변환된 다운로드 링크
- **예외 상황**: 
  - 파일 크기 초과 시 오류 (예: 50MB 제한)
  - 지원하지 않는 파일 형식 안내

#### 3.2.18 파일 다운로드 (뷰어)
- **설명**: 뷰어가 문서에 첨부된 파일을 다운로드
- **동작 방식**:
  - 문서 내 `::file-download` 영역에 다운로드 버튼 표시
  - 버튼 클릭 시 S3에서 파일 다운로드
  - 파일명과 아이콘 표시 (zip, pdf 등 형식별 아이콘)
- **입력**: 다운로드 버튼 클릭
- **출력/결과**: 파일 다운로드 시작
- **예외 상황**: 파일이 삭제된 경우 "파일을 찾을 수 없습니다" 표시

#### 3.2.19 공개 게시판
- **설명**: 공개 설정된 폴더와 문서를 트리형 목록으로 모아서 보여주는 페이지
- **동작 방식**:
  - 별도 공개 URL로 접근 (예: `https://pyulchim.com/board`)
  - 공개 설정된 폴더들이 좌측에 트리 구조로 표시
  - 폴더 클릭 시 해당 폴더의 공개 문서 목록 표시
  - 문서 클릭 시 문서 뷰어 페이지로 이동
  - 관리자 대시보드와 유사하나 관리 기능(삭제, 수정 등) 없음
- **입력**: 공개 게시판 URL 접근
- **출력/결과**: 공개된 폴더/문서 트리 목록
- **예외 상황**: 공개된 콘텐츠가 없을 경우 "공개된 문서가 없습니다" 표시

#### 3.2.20 공개 여부 설정
- **설명**: 폴더 또는 문서별로 공개 여부를 설정
- **동작 방식**:
  - **폴더 공개 설정**: 폴더를 공개하면 기본적으로 안의 모든 문서가 공개
  - **문서별 예외 설정**: 공개 폴더 안에서도 특정 문서는 비공개로 설정 가능
  - **비공개 폴더 내 공개 문서**: 비공개 폴더 안에서도 특정 문서만 공개 가능
  - 관리자 페이지에서 폴더/문서 옆에 공개 토글 버튼 표시
  - 공개 상태는 아이콘으로 표시 (🌐 공개 / 🔒 비공개)
- **입력**: 폴더 또는 문서 선택 후 공개 토글
- **출력/결과**: 공개 상태 변경, 공개 게시판에 반영
- **예외 상황**: 비밀번호 잠금된 문서는 공개 설정해도 비밀번호 입력 필요

#### 3.2.18 외부 콘텐츠 크래퍼 (개요)
- **설명**: 외부 서비스의 콘텐츠 링크를 입력하면 마크다운 문서로 자동 변환
- **동작 방식**:
  - 관리자 페이지에서 "링크로 추가" 버튼 클릭
  - 외부 링크 입력 (YouTube, LinkedIn, Threads)
  - 시스템이 자동으로 콘텐츠 추출 및 마크다운 변환
  - 변환된 마크다운 미리보기 제공
  - 확인 후 저장 시 일반 문서와 동일하게 관리
- **입력**: 외부 콘텐츠 URL
- **출력/결과**: 마크다운으로 변환된 문서
- **예외 상황**: 
  - 지원하지 않는 URL 형식 안내
  - 비공개 콘텐츠 접근 불가 시 오류 메시지
  - 크래핑 실패 시 재시도 또는 수동 입력 안내

#### 3.2.22 YouTube 크래퍼
- **설명**: YouTube 영상 링크를 마크다운 문서로 변환
- **동작 방식**:
  - YouTube URL 입력 (일반 링크, 단축 링크 모두 지원)
  - 추출 항목:
    - 영상 제목 → `::title`로 변환
    - 채널명, 업로드 날짜 → Frontmatter 메타데이터
    - 영상 설명 → 본문
    - 썸네일 이미지 → 이미지로 삽입
    - 임베드 코드 → iframe으로 삽입
    - **자막 추출** → 본문에 포함 (타임스탬프 옵션)
  - 자막 추출 옵션: 전체 자막 / 요약 / 타임스탬프 포함 여부 선택
- **입력**: YouTube URL, 자막 옵션
- **출력/결과**: 
  ```markdown
  ::title[영상 제목]
  
  ---
  source: https://youtube.com/watch?v=xxxxx
  channel: 채널명
  date: 2025-02-04
  ---
  
  ![썸네일](thumbnail_url)
  
  <iframe src="https://youtube.com/embed/xxxxx" ...></iframe>
  
  ## 영상 설명
  영상 설명 내용...
  
  ## 자막
  [00:00] 자막 내용...
  [00:30] 자막 내용...
  ```
- **예외 상황**: 
  - 자막이 없는 영상: "자막을 찾을 수 없습니다" 안내
  - 비공개/삭제된 영상: 접근 불가 오류

#### 3.2.23 LinkedIn 크래퍼
- **설명**: LinkedIn 포스트 링크를 마크다운 문서로 변환
- **동작 방식**:
  - LinkedIn 포스트 URL 입력
  - 추출 항목:
    - 작성자 이름, 직함 → Frontmatter 메타데이터
    - 게시 날짜 → Frontmatter
    - 포스트 본문 → 마크다운 본문
    - 첨부 이미지 → 이미지로 삽입
    - 원본 링크 → source 메타데이터
- **입력**: LinkedIn 포스트 URL
- **출력/결과**:
  ```markdown
  ::title[포스트 제목 또는 첫 줄]
  
  ---
  source: https://linkedin.com/posts/xxxxx
  author: 작성자 이름
  author_title: 직함
  date: 2025-02-04
  platform: LinkedIn
  ---
  
  포스트 본문 내용...
  
  ![이미지](image_url)
  ```
- **예외 상황**: 
  - 로그인 필요 콘텐츠: 접근 제한 안내
  - 삭제된 포스트: 오류 메시지

#### 3.2.24 Threads 크래퍼
- **설명**: Threads 포스트 링크를 마크다운 문서로 변환
- **동작 방식**:
  - Threads 포스트 URL 입력
  - 추출 항목:
    - 작성자 username → Frontmatter 메타데이터
    - 게시 날짜 → Frontmatter
    - 포스트 본문 → 마크다운 본문
    - 첨부 이미지/동영상 → 미디어로 삽입
    - 원본 링크 → source 메타데이터
- **입력**: Threads 포스트 URL
- **출력/결과**:
  ```markdown
  ::title[@username의 Threads]
  
  ---
  source: https://threads.net/@username/post/xxxxx
  author: @username
  date: 2025-02-04
  platform: Threads
  ---
  
  포스트 본문 내용...
  
  ![이미지](image_url)
  ```
- **예외 상황**: 
  - 비공개 계정: 접근 불가 안내
  - 삭제된 포스트: 오류 메시지

## 4. 사용자 시나리오

### 4.1 핵심 시나리오

**시나리오 1: 단일 문서 업로드 및 공유**
> 사용자: 관리자
> 목적: 작성한 마크다운 문서를 외부에 공유

1. 관리자 페이지에 로그인한다
2. 상단 또는 메인 영역의 "업로드" 버튼을 클릭한다
3. .md 파일을 선택하거나 드래그 앤 드롭한다
4. (선택) 폴더와 태그를 지정한다
5. 업로드 버튼을 클릭한다
6. 문서 목록에 새 문서가 추가된 것을 확인한다
7. COPY 버튼을 클릭하여 공유 URL을 복사한다
8. 공유 대상에게 URL을 전달한다

---

**시나리오 2: 이미지가 포함된 문서 업로드**
> 사용자: 관리자
> 목적: 로컬 이미지가 포함된 마크다운 문서 공유

1. 관리자 페이지에 로그인한다
2. "업로드" 버튼을 클릭한다
3. .md 파일을 선택한다
4. 시스템이 마크다운 내 로컬 이미지 경로를 감지한다
5. "이미지 파일도 함께 업로드하세요" 안내가 표시된다
6. 해당 이미지 파일들을 추가로 선택한다
7. 업로드 버튼을 클릭한다
8. 이미지가 S3에 업로드되고, 마크다운 내 경로가 자동 변환된다
9. 공유 URL로 접속하면 이미지가 정상적으로 표시된다

---

**시나리오 3: 여러 문서 배치 업로드**
> 사용자: 관리자
> 목적: 여러 마크다운 문서를 한 번에 업로드

1. 관리자 페이지에 로그인한다
2. "업로드" 버튼을 클릭한다
3. 여러 .md 파일을 동시에 선택한다 (Ctrl+클릭 또는 드래그)
4. (선택) 공통 폴더와 태그를 지정한다
5. 업로드 버튼을 클릭한다
6. 모든 문서가 순차적으로 업로드된다
7. 문서 목록에서 각 문서의 COPY 버튼으로 개별 URL을 복사한다

---

**시나리오 4: 문서에 비밀번호 설정**
> 사용자: 관리자
> 목적: 민감한 문서를 비밀번호로 보호

1. 관리자 페이지에서 문서 목록을 확인한다
2. 보호하려는 문서의 잠금 아이콘(🔒)을 클릭한다
3. 비밀번호 설정 모달이 표시된다
4. 비밀번호를 입력하고 확인한다
5. 잠금 아이콘이 활성화(노란색) 상태로 변경된다
6. 공유 URL을 전달받은 사람은 비밀번호를 입력해야 문서를 볼 수 있다

---

**시나리오 5: 잠긴 문서 열람 (뷰어)**
> 사용자: 뷰어 (공유 링크를 받은 사람)
> 목적: 비밀번호가 설정된 문서 열람

1. 공유받은 URL을 브라우저에서 연다
2. "이 문서는 비밀번호로 보호되어 있습니다" 화면이 표시된다
3. 전달받은 비밀번호를 입력한다
4. "확인" 버튼을 클릭한다
5. 비밀번호가 맞으면 문서 내용이 표시된다
6. (비밀번호가 틀리면 오류 메시지 표시 후 재입력)

---

**시나리오 6: 문서 검색 및 필터링**
> 사용자: 관리자
> 목적: 특정 문서를 빠르게 찾기

1. 관리자 페이지에 로그인한다
2. 좌측 폴더 목록에서 특정 폴더를 클릭하여 필터링한다
3. 또는 태그를 클릭하여 해당 태그의 문서만 표시한다
4. 또는 상단 검색창에 키워드를 입력하여 검색한다
5. 또는 시작일/종료일을 지정하여 기간 내 문서만 표시한다
6. 원하는 문서를 찾아 VIEW, COPY, DELETE 등 작업을 수행한다

---

**시나리오 7: 문서 삭제**
> 사용자: 관리자
> 목적: 더 이상 필요 없는 문서 삭제

1. 관리자 페이지에서 삭제할 문서를 찾는다
2. DELETE 버튼을 클릭한다
3. "정말 삭제하시겠습니까?" 확인 모달이 표시된다
4. "삭제" 버튼을 클릭한다
5. 문서가 목록에서 제거된다
6. 해당 문서의 공유 URL은 더 이상 접근 불가 (404)

---

**시나리오 8: 폴더/문서 공개 설정 및 공개 게시판 활용**
> 사용자: 관리자
> 목적: "이주의 AI 뉴스" 폴더를 공개하여 구독자들이 볼 수 있게 설정

1. 관리자 페이지에 로그인한다
2. 좌측 폴더 목록에서 "이주의 AI 뉴스" 폴더를 찾는다
3. 폴더 옆의 공개 토글(🌐)을 클릭하여 공개로 설정한다
4. 폴더 내 모든 문서가 기본적으로 공개 상태가 된다
5. (선택) 특정 문서는 비공개로 유지하고 싶다면 해당 문서의 공개 토글을 끈다
6. 공개 게시판 URL (`https://pyulchim.com/board`)을 구독자들에게 공유한다
7. 구독자들은 해당 URL에서 공개된 폴더와 문서 목록을 볼 수 있다

---

**시나리오 9: 공개 게시판 열람 (뷰어)**
> 사용자: 뷰어 (구독자)
> 목적: 공개된 AI 뉴스 및 LLM 리포트 열람

1. 공개 게시판 URL (`https://pyulchim.com/board`)에 접속한다
2. 좌측에 공개된 폴더 트리가 표시된다 (예: "이주의 AI 뉴스", "LLM 평가 리포트")
3. "이주의 AI 뉴스" 폴더를 클릭한다
4. 해당 폴더의 공개 문서 목록이 표시된다
5. 읽고 싶은 문서를 클릭한다
6. 문서 뷰어 페이지에서 내용을 열람한다
7. (비밀번호가 설정된 문서라면 비밀번호 입력 후 열람)

**시나리오 10: YouTube 영상을 마크다운으로 변환**

## 5. 화면 구성 (개요)

### 5.1 주요 화면 목록

| 화면명 | 설명 | 주요 요소 |
|-------|------|----------|
| 로그인 페이지 | 관리자 인증 | 아이디 입력, 비밀번호 입력, 로그인 버튼 |
| 관리자 대시보드 | 문서 관리 메인 화면 | 헤더, 좌측 사이드바(폴더/태그), 문서 목록, 검색/필터 |
| 업로드 모달 | 문서 업로드 인터페이스 | 파일 선택 영역, 폴더 선택, 태그 선택, 업로드 버튼 |
| 링크 추가 모달 | 외부 링크 크래퍼 인터페이스 | URL 입력, 옵션 선택, 미리보기, 저장 버튼 |
| 문서 뷰어 페이지 | 공유 URL로 접근하는 문서 열람 화면 | 툴바(다크모드, 복사, 저장, 목차), 본문 영역, 목차 사이드바 |
| 공개 게시판 | 공개된 폴더/문서 트리 목록 | 폴더 트리, 문서 목록, 검색 (관리 기능 없음) |
| 비밀번호 입력 페이지 | 잠긴 문서 접근 시 | 안내 문구, 비밀번호 입력, 확인 버튼 |
| 404 페이지 | 존재하지 않는 문서 | 오류 안내, 홈으로 이동 버튼 |

### 5.2 화면 흐름 (간략)

**관리자 흐름:**
```
로그인 → 관리자 대시보드 → 업로드 모달 → 대시보드 (목록 갱신)
                         → 링크 추가 모달 → 미리보기 → 저장 → 대시보드
                         → 문서 VIEW → 문서 뷰어 페이지
                         → 문서 DELETE → 삭제 확인 → 대시보드
                         → 공개 설정 토글 → 공개 게시판에 반영
                         → LOGOUT → 로그인
```

**뷰어 흐름:**
```
공유 URL 접근 → (잠금 문서인 경우) 비밀번호 입력 → 문서 뷰어 페이지
            → (잠금 아닌 경우) 문서 뷰어 페이지
            → (존재하지 않는 경우) 404 페이지

공개 게시판 접근 → 폴더 트리 탐색 → 문서 선택 → 문서 뷰어 페이지
```

### 6.2 커스텀 디렉티브 문법 상세

커스텀 디렉티브 문법을 지원합니다.

#### 6.2.1 인라인 디렉티브 (`::name[내용]{속성}`)

| 디렉티브 | 문법 | 설명 | 렌더링 |
|----------|------|------|--------|
| title | `::title[제목 텍스트]` | 문서 메인 제목 | 노란색 배경의 큰 제목 박스 |
| section-title | `::section-title[섹션명]` | 섹션 구분 제목 | 파란색 배경의 섹션 헤더 |
| file-download | `::file-download[파일명]{url="/path/file.zip"}` | 다운로드 링크 | 파일 아이콘 + 클릭 가능한 링크 |
| toc | `::toc[목차]` | 목차 생성 | 문서 내 헤딩 기반 목차 리스트 |

#### 6.2.2 블록 디렉티브 (`:::name{속성}`)

| 디렉티브 | 문법 | 설명 | 렌더링 |
|----------|------|------|--------|
| details | `:::details{summary="제목"}` | 접기/펼치기 | 클릭하면 펼쳐지는 영역 |
| info | `:::info` | 정보 안내 | 파란색 테두리 정보 박스 |
| warning | `:::warning` | 경고 안내 | 주황색 테두리 경고 박스 |
| tip | `:::tip` | 팁/권장사항 | 초록색 테두리 팁 박스 |

#### 6.2.3 디렉티브 사용 예시

**제목 디렉티브:**
```markdown
::title[박준의 CLAUDE CODE 설정 따라하기]
```
→ 노란색 배경의 메인 제목으로 렌더링

**메타데이터 (Frontmatter):**
```yaml
---
date: 2026-01-15
claude_code_version: 2.1.7
os: macOS (일부 기능은 macOS 전용)
---
```
→ 초록색 배경의 메타데이터 박스로 렌더링 (DATE, CLAUDE_CODE_VERSION, OS 각각 표시)

**섹션 제목:**
```markdown
::section-title[설정 파일 다운로드]
```
→ 파란색 배경의 섹션 헤더로 렌더링

**파일 다운로드:**
```markdown
::file-download[claude-code-settings.zip]{url="/files/claude-code-settings.zip"}
```
→ 📎 아이콘과 함께 클릭 가능한 다운로드 링크로 렌더링

**목차:**
```markdown
::toc[목차]
```
→ 문서 내 모든 헤딩을 스캔하여 클릭 가능한 목차 리스트 생성

**접기/펼치기:**
```markdown
:::details{summary="CLAUDE.md 전체 보기"}
여기에 숨겨진 내용...
:::
```
→ "CLAUDE.md 전체 보기" 클릭 시 내용이 펼쳐짐

### B. 마크다운 렌더링 기술 참고사항

마크다운 렌더링 구현 시 참고할 수 있는 라이브러리 및 구현 방식입니다.

#### B.1 권장 기술 스택

| 라이브러리 | 용도 | 비고 |
|-----------|------|------|
| `react-markdown` | 마크다운 → React 변환 | 핵심 렌더링 라이브러리 |
| `remark-gfm` | GFM 확장 문법 | 테이블, 체크리스트, 취소선 |
| `rehype-raw` | HTML 태그 허용 | `<span>`, `<figure>` 등 |
| `rehype-highlight` | 코드 구문 강조 | 또는 `react-syntax-highlighter` |
| `remark-directive` | 커스텀 디렉티브 | `::title`, `:::info` 등 |
| `remark-math` + `rehype-katex` | 수식 렌더링 | LaTeX 문법 |
| `gray-matter` | Frontmatter 파싱 | YAML 메타데이터 추출 |

#### B.2 react-markdown 기본 지원 범위

다음 문법은 react-markdown이 기본 또는 플러그인으로 지원하므로, **커스텀 컴포넌트로 스타일링만 하면 됨**:

| 문법 | 지원 방식 |
|------|----------|
| 제목, 굵게, 기울임, 취소선 | 기본 지원 |
| 목록, 링크, 이미지 | 기본 지원 |
| 인라인 코드, 코드 블록 | 기본 지원 |
| 인용구, 수평선 | 기본 지원 |
| 테이블, 체크리스트 | `remark-gfm` 플러그인 |
| HTML 태그 렌더링 | `rehype-raw` 플러그인 |
| 코드 구문 강조 | `rehype-highlight` 플러그인 |
| 수식 (LaTeX) | `remark-math` + `rehype-katex` |

#### B.3 커스텀 구현 필요 항목

다음 문법은 **커스텀 로직 또는 플러그인 확장**이 필요함:

| 문법 | 구현 방법 |
|------|----------|
| `::title[텍스트]` | `remark-directive` + 커스텀 컴포넌트 |
| `::section-title[텍스트]` | `remark-directive` + 커스텀 컴포넌트 |
| `::file-download[파일]{url="..."}` | `remark-directive` + 커스텀 컴포넌트 |
| `::toc[목차]` | 헤딩 파싱 후 목차 생성 로직 |
| `:::info`, `:::warning`, `:::tip` | `remark-directive` + 커스텀 컴포넌트 |
| `:::details{summary="..."}` | `remark-directive` + 커스텀 컴포넌트 |
| Twitter 임베드 | `rehype-raw` + Twitter 위젯 스크립트 로드 |
| 이미지 캡션 (`*텍스트*`) | 커스텀 remark 플러그인 또는 후처리 |
| Frontmatter 메타데이터 박스 | `gray-matter`로 파싱 후 별도 컴포넌트로 렌더링 |

#### B.4 구현 예시 코드

```jsx
import ReactMarkdown from 'react-markdown'
import remarkGfm from 'remark-gfm'
import remarkDirective from 'remark-directive'
import rehypeRaw from 'rehype-raw'
import rehypeHighlight from 'rehype-highlight'

// 커스텀 컴포넌트 (스타일링)
const components = {
  h1: ({children}) => (
    <h1 className="text-3xl font-bold bg-yellow-200 p-4 rounded">{children}</h1>
  ),
  h2: ({children}) => (
    <h2 className="text-2xl font-bold bg-blue-400 text-white p-3 rounded">{children}</h2>
  ),
  blockquote: ({children}) => (
    <blockquote className="border-l-4 border-yellow-400 pl-4 italic">{children}</blockquote>
  ),
  code: ({inline, className, children}) => {
    if (inline) {
      return <code className="bg-gray-100 px-1 rounded">{children}</code>
    }
    return (
      <div className="relative">
        <pre className={className}><code>{children}</code></pre>
        <button className="absolute top-2 right-2">COPY</button>
      </div>
    )
  },
  // ... 기타 컴포넌트
}

function MarkdownViewer({ content }) {
  return (
    <ReactMarkdown
      remarkPlugins={[remarkGfm, remarkDirective]}
      rehypePlugins={[rehypeRaw, rehypeHighlight]}
      components={components}
    >
      {content}
    </ReactMarkdown>
  )
}
```

#### B.5 구현 우선순위 제안

| 단계 | 구현 내용 |
|------|----------|
| 1단계 | react-markdown + remark-gfm + rehype-raw + rehype-highlight 기본 설정 |
| 2단계 | 커스텀 컴포넌트로 스타일링 (색상, 배경, 여백 등) |
| 3단계 | gray-matter로 Frontmatter 파싱 및 메타데이터 박스 구현 |
| 4단계 | remark-directive로 커스텀 디렉티브 (::title, :::info 등) 구현 |
| 5단계 | 목차 자동 생성, Twitter 임베드 등 부가 기능 |
